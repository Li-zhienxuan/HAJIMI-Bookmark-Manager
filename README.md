# HAJIMI 书签管理器 (CNB & GitHub 版)

HAJIMI 是一个基于文件的“本地优先”书签管理工具。它通过 CNB (腾讯云原生研发平台) 或 GitHub 的 API，将书签数据同步至您个人的 Git 仓库中，实现数据的完全自主掌控。

## 一、 项目简介

本应用旨在解决传统书签管理器依赖第三方云服务、数据不透明以及跨境访问速度慢的问题。通过将 Git 仓库作为后端存储，您的每一条书签都以 JSON 格式安全地存储在您自己的代码库中。

## 二、 核心特性

1. **数据所有权**：您的数据即代码库中的 `bookmarks.json` 文件，不经过任何中转服务器。
2. **多端同步**：支持 CNB (国内极速) 和 GitHub (全球通用) 双服务商同步。
3. **隐私安全**：所有 API 访问令牌 (Token) 仅存储在浏览器本地，保障账户安全。
4. **离线优先**：即便在无网络环境下，依然可以流畅检索和浏览本地缓存的书签。
5. **兼容性强**：支持主流浏览器 (Chrome, Edge, Firefox) 的 HTML 书签导入。

## 三、 快速上手 (推荐 CNB 方案)

对于国内用户，推荐使用 CNB 以获得极速的同步体验。

1. **注册与登录**：访问 [cnb.cool](https://cnb.cool)，使用微信、QQ 或手机号登录。
2. **创建数据仓库**：新建一个 Git 仓库（例如命名为 `my-bookmarks`），建议将其设为“私有”。
3. **生成访问令牌 (Token)**：
   - 进入 [个人设置] -> [访问令牌]。
   - 创建新令牌，并确保勾选 `repo` 权限。
4. **在应用中配置**：
   - 进入 HAJIMI 的“同步与设置”页面。
   - 服务商选择 **CNB**。
   - 输入您的用户名、仓库名以及生成的 Token。
   - 点击“保存配置”后点击“立即同步”。

## 四、 常见问题与替代方案

1. **没有 GitHub 账号怎么办？**
   - 方案一：使用 CNB。CNB 是国内访问最快的 Git 服务之一，环境纯净且无门槛。
   - 方案二：纯离线模式。您可以不配置任何同步信息，书签将保存在浏览器本地存储中。
   - 方案三：手动备份。定期在设置中使用“导出备份 (JSON)”功能，将数据手动保存在本地磁盘。

2. **如何迁移现有书签？**
   - 在浏览器书签管理器中导出为 HTML 文件，然后在 HAJIMI 的设置中选择“从浏览器导入 (HTML)”即可一键完成迁移。

## 五、 技术实现

1. **前端框架**：React 19, TypeScript
2. **样式处理**：Tailwind CSS
3. **图标方案**：Lucide React
4. **存储逻辑**：浏览器 LocalStorage + Git REST API (OpenAPI)
